# extensions 模块设计

## 模块定位
- 负责 MCP、Skills、Plugin 等扩展能力接入并统一适配到工具契约。
- 不负责主循环与任务调度控制。

## 内部实现逻辑
- 扫描并加载扩展来源。
- 将外部能力映射为统一工具适配器。
- 管理扩展能力元数据与生命周期。

## 对外契约
- 暴露扩展加载器与能力提供者接口。
- 输入：扩展配置与上下文。
- 输出：可注册的工具适配器集合。

## 依赖关系
- 允许依赖：扩展协议层、工具抽象。
- 禁止依赖：`agent` 内部状态读写。

## 可观测性
- 扩展加载成功率、加载耗时、失败原因。
- 扩展工具调用审计。

## 测试策略
- 扩展加载契约测试。
- 协议适配兼容性测试。
- 异常扩展隔离测试。
