# Doc Coauthoring

## 何时使用

- 用户只有碎片化想法、口述片段、提纲或批注。
- 用户希望把草稿整理成结构完整、可继续协作的正式文档。

## 工作方式

- 先把原始材料归类为背景、目标、论点、证据、待确认项。
- 优先给出目录，再逐段补全文案，不要一次性堆成长文。
- 对信息空缺处明确标记“待补充”，不要擅自虚构事实。

## 交付要求

- 输出应包含标题层级、关键论点和后续协作建议。
- 草稿要便于继续编辑，而不是追求一次性定稿。
