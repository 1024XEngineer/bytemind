# storage 模块设计

## 模块定位
- 负责会话、任务、审计数据的文件存储、恢复回放与幂等去重。
- 不负责业务决策和任务调度。

## 内部实现逻辑
- 采用 append-only 写入。
- 通过原子落盘和文件锁保证一致性。
- 通过事件 ID 去重保障恢复幂等。

## 对外契约
- 暴露会话事件追加、回放与任务日志写入接口。
- 输入：会话 ID/任务 ID、事件数据。
- 输出：写入结果、回放结果、标准错误。

## 依赖关系
- 允许依赖：文件系统抽象、锁机制。
- 禁止依赖：`agent`、`tools`、`runtime` 内部实现。

## 可观测性
- 写入吞吐、回放耗时、锁等待时长。
- 写入失败与恢复失败审计。

## 测试策略
- 崩溃恢复测试。
- 并发写一致性测试。
- 回放幂等测试。
