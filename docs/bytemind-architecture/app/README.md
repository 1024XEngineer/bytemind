# app 模块设计

## 模块定位
- 负责系统启动装配、配置注入和生命周期管理。
- 不负责业务编排、策略判断和工具执行。

## 边界
- 做：组件装配、配置加载、进程生命周期托管、依赖注入。
- 不做：业务主循环、权限决策、工具执行、任务调度。

## 内部实现逻辑
- 读取配置与运行模式。
- 构建依赖图并注入 `agent`、`tools`、`policy`、`storage` 等组件。
- 托管进程级生命周期（启动、优雅退出、资源释放）。

## 对外契约
- 暴露应用运行与关闭接口。
- 输入：`context.Context`。
- 输出：标准 `error`，错误需可观测可追踪。

## 依赖关系
- 允许依赖：配置、日志、`agent` 入口接口。
- 禁止依赖：具体工具实现、策略实现细节、存储底层实现。

## 可观测性
- 启停日志、启动耗时、依赖装配失败原因。
- 进程级健康状态与退出原因。

## 测试策略
- 启动/关闭生命周期测试。
- 依赖注入失败路径测试。
- 配置异常与恢复策略测试。
