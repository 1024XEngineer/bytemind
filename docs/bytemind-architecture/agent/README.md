# agent 模块设计

## 模块定位
- 负责主循环：消息处理、上下文拼装、模型交互、工具调用编排。
- 不负责工具实现细节、权限规则实现和底层存储实现。

## 边界
- 做：主循环编排、模型事件处理、工具调用路由、结果归并。
- 不做：工具内部执行、权限规则实现、任务系统实现、存储实现。

## 内部实现逻辑
- 接收用户输入并加载会话上下文。
- 组装系统提示词、模式提示词与运行时上下文。
- 流式消费模型输出并驱动工具调用。
- 汇总工具结果并返回用户可消费事件。

## 对外契约
- 暴露用户消息处理接口，返回事件流。
- 输入：`sessionID`、用户输入、`context.Context`。
- 输出：事件流与语义化错误。

## 依赖关系
- 允许依赖：`tools`、`runtime`、`policy`、`storage` 的抽象接口。
- 禁止依赖：各模块具体实现与 provider 私有类型。

## 可观测性
- 每轮推理耗时、工具调用次数、失败原因分布。
- 关键状态转移审计事件。

## 测试策略
- 主循环单测。
- 工具调用编排契约测试。
- 上下文压缩与超长重试场景测试。
