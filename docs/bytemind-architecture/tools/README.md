# tools 模块设计

## 模块定位
- 负责工具契约、注册、参数校验、执行调度和统一事件流输出。
- 不负责会话级策略判断与业务编排。

## 边界
- 做：工具 schema 校验、执行编排、统一事件流、超时/取消/重试语义传递。
- 不做：权限规则决策、会话主循环管理、存储实现。

## 内部实现逻辑
- 加载内置工具与扩展工具。
- 对输入参数做 schema 校验。
- 执行工具并将结果统一封装为标准事件流。

## 对外契约
- 暴露工具注册表与工具执行接口。
- 输入：工具名、参数、调用上下文。
- 输出：统一事件流（`start/chunk/result/error`）。

## 依赖关系
- 允许依赖：`policy` 抽象、`storage` 抽象、扩展适配层。
- 禁止依赖：`agent` 内部状态。

## 可观测性
- 工具调用成功率、失败率、耗时分布。
- 高风险工具触发审计。

## 测试策略
- 参数校验契约测试。
- 事件流顺序与幂等测试。
- 超时/取消/重试语义测试。
