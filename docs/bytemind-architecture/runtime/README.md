# runtime 模块设计

## 模块定位
- 负责任务系统、状态机、并发调度和多代理协作。
- 不负责权限规则定义与业务策略定义。

## 边界
- 做：任务状态机、取消传播、重试控制、任务日志增量读取、子代理调度归并。
- 不做：权限策略定义、工具参数校验、会话存储细节。

## 内部实现逻辑
- 管理任务状态流转：`pending -> running -> terminal`。
- 提供超时、取消传播和重试机制。
- 支持同步/异步子代理执行与结果归并。

## 对外契约
- 暴露任务提交、取消、等待、读取输出的能力。
- 输入：任务请求与执行上下文。
- 输出：任务句柄、任务结果、执行日志流。

## 依赖关系
- 允许依赖：`policy` 抽象、`storage` 抽象。
- 禁止依赖：`agent` 具体实现。

## 可观测性
- 任务成功率、平均完成时长、重试次数。
- 任务状态转移审计与异常终止原因。

## 测试策略
- 状态机单测。
- 并发与取消传播测试。
- 子代理归并一致性测试。
