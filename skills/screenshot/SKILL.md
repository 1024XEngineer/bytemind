# Screenshot

## 何时使用

- 用户要截图留痕、反馈问题、制作操作说明或培训材料。
- 用户需要明确截哪里、截几张、每张图表达什么。

## 工作方式

- 先确认截图对象、步骤顺序、是否含敏感信息、保存路径和命名规则。
- 当前项目不再内置平台脚本；优先使用系统、IDE 或专用工具的截图能力。
- 生成说明文案时，按“步骤编号 + 截图意图 + 标注文字”组织内容。

## 交付要求

- 每张截图都要有目的、文件名和对应说明。
- 涉及隐私、密钥、账号或客户数据时，先要求打码或裁切。
